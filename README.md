# 电商全栈系统 - 接口自动化测试实战项目

本项目是针对电商全栈系统（包含移动端商城与 Web 管理后台）进行的接口自动化测试实践。通过本项目，我深入实践了接口测试生命周期管理、测试框架搭建及数据持久化校验。

## 🌟 实践目标
- **业务梳理**：独立梳理电商核心链路（SKU、购物车、结算、订单状态流转）。
- **框架搭建**：使用 Python + Requests + Pytest 搭建工程化测试框架。
- **质量保证**：实现全链路业务闭环测试，确保前端操作与后端数据库变动的一致性。

## 🛠 技术栈
- **测试框架**: Pytest (结合 Fixture 注入进行环境初始化)
- **网络协议**: Requests (封装底层接口调用)
- **数据驱动**: JSON (实现测试数据与脚本分离)
- **数据库校验**: PyMySQL (用于校验后端存储与事务提交)
- **测试报告**: Allure (记录详细执行日志与测试结果)

## 📁 项目核心设计

### 1. 接口对象模型 (API-Object Pattern)
参照 POM 思想，将各业务模块（认证、地址、购物车、商品、结算）封装为独立的类，实现业务逻辑与测试脚本的解耦，大幅提升代码维护性。

### 2. 数据驱动测试 (DDT)
测试用例参数通过外部 `data/` 目录下的 JSON 文件透传，支持在不修改代码的情况下快速扩展异常场景覆盖（如非法参数、边界值校验）。

### 3. 环境幂等性保障
利用 Pytest Fixture 实现：
- **前置准备**：自动化获取 Session 令牌。
- **后置清理**：测试结束后自动回滚/物理删除测试产生的脏数据（如生成的地址记录、购物车商品），确保环境可多次重复运行。
